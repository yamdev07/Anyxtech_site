# Plan: Redesign Dashboard Footer — Full-Width, Rich Content

## Problem
Current footer is a small glass bar with only copyright, theme switcher, and a few nav links. It looks cramped on login page and doesn't provide useful admin navigation context. User wants it to match the richness of the site footer.

## Goal
A full-width, multi-column footer that mirrors the sidebar navigation structure, includes contact info from site settings, and works on both dashboard and auth pages.

## Design

### Layout (3-column grid on desktop, stacked on mobile)
```
┌──────────────────────────────────────────────────────────────────┐
│  AnyxTech     Navigation        Contenu           Contact       │
│  Description  Vue d'ensemble    Services          Address       │
│  Socials      Analytique        Offres            Phone         │
│               Messages          Partenaires       Email         │
│               Paramètres        Actualités        Hours         │
│                                  Témoignages                    │
├──────────────────────────────────────────────────────────────────┤
│  © 2026 AnyxTech Bénin     [Theme: Clair | Sombre | Système]   │
│  Conçu avec passion à Cotonou    [Déconnexion]                  │
└──────────────────────────────────────────────────────────────────┘
```

### Key decisions
- **Full-width**: Remove `max-w-7xl` constraint, use `w-full` + horizontal padding
- **No dynamic island**: Not a floating pill — a solid full-width bar
- **Dark glass background**: Same `rgba(30,33,43,0.85)` with blur, works in both themes
- **Site settings**: Fetch contact info server-side, pass as props (since footer is client component)
- **Auth pages**: Same footer, just hide user-specific info (email, logout)
- **Sidebar offset**: `lg:pl-64` only when sidebar is present

## Files to modify

### 1. `components/dashboard/DashboardFooter.tsx` (rewrite)
- Accept props: `email`, `noSidebar`, `siteSettings` (contact info)
- 3-column grid: Brand+Socials | Navigation+Contenu | Contact
- Bottom bar: copyright + theme switcher + logout
- Full-width, no max-w constraint

### 2. `app/(dashboard)/layout.tsx`
- Pass site settings to footer via `getSiteSettings()`

### 3. `app/(auth)/layout.tsx`
- Add footer with `noSidebar` prop and site settings

### 4. `app/(auth)/dashboard/login/page.tsx`
- Remove the footer (it's now in the auth layout)

## Verification

- Visit `/dashboard/login` — same footer, no sidebar offset, no user info
- Toggle theme — footer adapts
- Check mobile — columns stack properly
